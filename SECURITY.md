# 安全策略

## 🔒 报告安全漏洞

如果你发现了安全漏洞，**请不要公开提交 Issue**。

### 📧 如何报告

请发送邮件至：[你的邮箱地址]

邮件内容请包含：

- 漏洞描述
- 受影响的版本
- 重现步骤
- 可能的影响范围
- 如果你已经想到了修复方案，也请一并说明

### ⏰ 响应时间

我们会在 **48 小时内** 确认收到你的报告，并在 **7 天内**给出修复计划。

### 🔐 保密承诺

- 我们会尽可能快地确认收到你的报告
- 我们会与你保持沟通，了解漏洞详情
- 我们会尽快修复漏洞
- 我们会在修复后公开致谢（如果你同意）

## 🛡️ 安全最佳实践

### 作为用户

1. **保持更新**: 定期更新到最新版本
2. **验证来源**: 只从官方渠道下载
3. **检查权限**: 应用只请求必要的权限

### 作为开发者

1. **代码审查**: 所有代码必须经过审查
2. **依赖审计**: 定期更新依赖包
3. **最小权限**: 应用遵循最小权限原则
4. **数据保护**: 敏感数据妥善处理

## 🏆 安全致谢

我们感谢所有通过负责任的方式报告安全漏洞的研究者。

公开报告的安全漏洞会在 [Security Advisories](https://github.com/Chee-0806/etools/security/advisories) 中发布。
